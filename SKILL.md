# 北航结课作业 / 课程论文辅助技能

## 目标

**辅助**用户根据北航课程结课论文/课程考核模板准备 Word 文档。重点不是”代替用户写论文”，而是”帮用户按老师给的模板快速搭建框架”：保留模板版式、Logo、页边距、段落样式等格式要素，提供结构化的内容草稿供用户审核、修改和完善。**用户需要对最终内容负责，自行审查、修改并确认后再提交。**

## 运行过程提示

执行本技能时，应在每一步开始前简短告诉用户当前正在做什么（不需要说具体内容，只说步骤）。

**标准提示话术：**

1. 收集信息阶段：`正在收集论文基本信息（课程、姓名、学号、要求等）`
2. 读取参考资料：`正在读取你上传的课件/要求/参考资料`
3. 选题/大纲阶段：`正在根据课程要求拟定题目和结构`
4. 写作阶段：`正在撰写论文正文，约需要1-2分钟`
5. 去套话/去 AI 痕迹：`正在进行学术语言优化和套话清理`
6. 套用模板生成 Word：`正在按模板格式生成 DOCX 文件`
7. 最终自检：`正在进行格式和内容自检`

每完成一个关键阶段，也可以用简洁的话术确认，例如：
- `已完成：模板样式分析`
- `已完成：10阶学术语言优化`
- `已完成：文档生成 + 自检通过`

## 启动时的引导式信息收集

技能触发后，先判断已有信息够不够启动生成。不必等所有信息齐全再开始。

用引导式提问，一次性列出还缺的信息，不要像填表格一样压迫感太强。用户没有提供的项，后续先用占位符填入，并在交付时提醒用户需要补哪些信息。

推荐的引导式提问示例：

> 我帮你按北航模板生成结课论文，需要你填几个信息，不知道的可以先空着或让我用占位：
> - 课程名：
> - 姓名：
> - 学号：
> - 学院/专业/班级：
> - 老师：
> - 字数要求（如 3000 字）：
> - 主题或选题范围（我也可以帮你推荐）：
> - 是否需要 AI 使用披露：
> - 有课件、老师要求、参考论文、草稿的话也可以一并发来，我会按资料内容写。

如果用户说“你先帮我选题/其他你定”，可以自行推荐题目和结构；但姓名、学号等个人信息不要编造，会用占位符标出待补。

用户回答后，直接基于已有信息生成；等用户补充新信息后再增量更新文档即可。

## 参考资料上传与头脑风暴

在选题、拟大纲或确定写作方向时，明确告诉用户可以上传参考资料。可接受的资料包括：

- 老师发的作业要求、评分标准、课程论文模板；
- 课件 PDF、教材章节、课堂笔记；
- 指定阅读文献、参考论文、政策文件；
- 用户已有草稿、摘要、提纲、读书笔记；
- 老师强调过的关键词、课堂讨论主题或考试范围。

处理原则：

1. 如果用户已经上传资料，先读取并提炼要求、主题边界、关键词、可引用观点，再推荐选题或大纲。
2. 如果用户还没有上传资料，在头脑风暴前提醒一句：可以把课件、老师要求、参考论文或已有草稿发来，我会优先按资料内容选题；如果暂时没有，也可以直接基于课程名和作业要求推荐。
3. 推荐选题时说明每个题目为什么适合：对应哪些课程材料、能引用哪些资料、风险在哪里、是否容易写够字数。
4. 生成正文时优先使用用户资料中的概念、案例、政策表述和参考文献，不要凭空说“老师课件提到”。
5. 如果资料不足以支撑某个选题，应说明缺口，并建议用户补充哪类材料。

## 工作流

### 1. 读取材料和要求

1. 找到用户提供的模板和要求文件，优先级：
   - `.docx` 模板：用于最终输出格式。
   - `.pdf` 考核说明：用于提取字数、结构、命名、截止时间、引用要求。
   - 课件/教材/笔记：用于限定选题和论证范围。
   - 若都没有模板，使用内置模板。
2. 对 `.docx` 模板使用 DOCX 解析方式读取正文。不要只凭文件名猜要求。
3. 提炼并向用户确认关键信息：课程名、题目/选题范围、字数下限、必备结构、引用规范、AI 使用披露要求、文件命名要求、截止时间。

### 2. 选题和大纲

如果用户没有题目，推荐一个稳妥题目，并说明为什么适合：

- 贴合课程范围
- 有现实问题意识
- 容易展开到字数要求
- 容易加入规范引用

避免选过于宏大、空泛、宣传口号式题目。对思政/马克思主义方法论类课程，优先选择”具体社会问题 + 方法论视角”的题型。对公共管理、健康治理、公共政策类课程，优先选择”现实政策问题 + 治理机制/路径分析”的题型。

### 3. 生成论文初稿

根据选题和大纲，生成完整论文初稿（**此时不考虑语言优化，先生成内容**）：

- 结构完整：题目、摘要、关键词、引言、正文、结论、参考文献；若模板另有要求，以模板为准。
- 摘要约 200 字，说明研究缘由、对象、方法、内容和结论。
- 关键词 3–5 个。
- 引言要提出问题、说明意义、简述研究现状或讨论背景，并交代文章结构。
- 正文围绕问题展开，不堆概念。
- 结论回扣问题，不写成口号。
- 参考文献与正文引用编号对应，如 `[1]`、`[2]`。
- 若用户要求或模板要求披露 AI，文末加入”人工智能使用披露”；若无明确要求，则**不加入**。

**输出：** 初稿文本（纯文本，尚未套用模板）

### 4. 优化论文语言（11 阶流水线）

**强制要求：** 初稿生成后，**必须自动运行**以下 3 阶段、11 步骤进行学术化审计，不能跳过。优化完成后再套用模板。

**优化目标：** 使文本达到：概念准确、逻辑清楚、语气克制、结构稳定、术语一致、格式规范。

#### 一阶处理：去口语、去套话、去主观化

1. 口语正式化
   - 识别口语词、网络化表达、随意化搭配和非正式句式。
   - 替换为准确、自然、符合学术语境的中文书面语。
   - 保留原文核心含义，不增加新的观点或判断。
   - 不使用生僻、古奥或过度文言化的表达。

2. 套话清理
   - 删除或替换“众所周知”“不可否认”“值得注意的是”“总而言之”“综上所述”“显而易见”等高频套话。
   - 拆解机械排比、强行递进和对仗式结构。
   - 若过渡词没有实际逻辑功能，应直接删除。
   - 若确有逻辑关系，应使用简洁、准确的学术衔接方式。

3. 客观转写
   - 识别“极其重要”“充分证明”“显然”“完美解释”等主观或绝对化表达。
   - 将第一人称评价转化为第三人称学术观察。
   - 保持论证力度，但降低情绪色彩。
   - 不削弱原文基于材料或数据形成的有效判断。
   - 避免使用过度确定性的断言。

#### 二阶重构：重建句法结构，理清句间逻辑

4. 长句拆分
   - 识别多重从句、过度嵌套、主语不明和修饰关系混乱的句子。
   - 在保留信息完整性的前提下，将长句拆分为若干清晰句子。
   - 明确每个句子的主语、谓语和逻辑关系。
   - 不遗漏原文中的变量、对象、时间、条件或数据。
   - 避免拆分后形成碎片化、缺乏衔接的句群。

5. 逻辑衔接
   - 判断句间关系属于因果、转折、递进、并列、解释、比较还是总结。
   - 补充必要的逻辑连接词。
   - 删除冗余、重复或误导性的连接词。
   - 不强加原文并未支持的因果关系。
   - 不为了“显得学术”而堆砌连接词。

6. 适度名词化
   - 识别动词密集、口语化或叙述性过强的表达。
   - 将其改写为适当的名词短语或抽象概念表达。
   - 提升文本的信息密度和学术节奏。
   - 不过度名词化，避免制造晦涩、僵硬的表达。
   - 例如：“因为人们迁移到了城市”→“伴随人口向城市迁移”；“研究者对材料进行了分析”→“对材料的分析表明”。

#### 三阶校准：术语准确、表达精简、重点清楚

7. 术语校准
   - 识别社会学、定性研究、传播学或相关人文社科文本中的术语表达。
   - 将含混、泛化或不规范的说法调整为更准确的学科术语。
   - 保持研究方法和理论框架本身不被改变。
   - 不虚构理论来源、学者名称或研究路径。
   - 例如：”聊天资料”→”访谈材料”；”把内容分类”→”编码”；”找出背后的主题”→”主题分析”。

8. 冗余压缩
   - 删除重复形容词、同义反复和无效修饰。
   - 压缩不承担论证功能的句子。
   - 保留必要的概念、数据、语境和方法信息。
   - 不删除影响研究语义的限定条件。
   - 例如：”具有非常重要的现实意义”→”具有现实意义”；”进行了一个比较深入的分析”→”进行了深入分析”。

9. 格式规范化
   - 去掉中文与英文、数字之间的多余空格：如”使用 Python 语言”→”使用Python语言”；”共 100 份问卷”→”共100份问卷”。
   - 检查英文单词、数字前后是否有不必要的空格。
   - 确保标点符号使用一致：中文用中文标点，英文用英文标点。
   - 例如：”使用 Python 语言”→”使用Python语言”；”共 100 份”→”共100份”。

10. 重点调整
    - 识别过度突出研究者行为的句子。
    - 将句子重点转向研究对象、资料、方法或发现。
    - 必要时使用”由””被””基于””通过”等结构，但避免翻译腔。
    - 不机械使用被动语态。
    - 例如：”我们分析了访谈资料”→”对访谈资料的分析显示”；”研究者发现受访者存在差异”→”受访者之间呈现出差异”。

#### 最终审计

10. 语义核验
    - 对比原始文本与改写文本，确认语义、数据、概念和逻辑关系是否被完整保留。
    - 检查改写后是否遗漏原文信息。
    - 检查是否改变研究对象、理论判断、方法描述或数据含义。
    - 检查是否引入原文没有的观点、证据或结论。
    - 对语法、标点和格式问题进行最终修正。
    - 若发现语义偏差，应明确标注并修正。

#### 输出标准

完成整套审计后，最终文本应达到：
- 保持中文母语者的自然表达，不制造翻译腔。
- 风格严谨、客观、克制。
- 不使用夸张修辞。
- 不使用机械排比。
- 不改变原文核心意思。
- 如果论文篇幅较长或改动点较多，在交付时列出关键修改说明。

### 5. 套用 Word 模板

**使用步骤4优化后的文本**，套用模板生成最终文档。核心原则：最终文件必须是真正基于 `.docx` 模板生成，而不是新建一个看起来相似的文档。**完全保留模板段落设置，不做额外覆盖。**

如果模板中已有明确的字体、字号、行距、页边距设置，严格按模板执行；模板不明确时，使用以下论文标准格式：
- **标题**：黑体三号（对应 Word 内部 32 半磅单位）
- **摘要、关键词、正文**：宋体小四（对应 Word 内部 24 半磅单位），1.5 倍行距
- **页边距**：常规设置（上下约 2.54 cm，左右约 3.17 cm，或模板默认值）
- **日期**：自动使用当前系统日期，格式为”YYYY年MM月DD日”，填入封面表格 Row 6

应做到：

- 复制原 `.docx` 包结构。
- 保留 `word/media/` 中的图片、Logo、校徽等资源。
- 保留 `word/_rels/document.xml.rels` 中的图片关系。
- 保留页面设置、页边距、段落样式、字体、行距、首行缩进。
- **完全保留模板段落的 pPr**（pStyle、spacing、ind），只在 run 级别修改文本内容，不在段落级别强制覆盖样式或格式。
- 删除模板提示文字和示例正文，不得残留旧论文题目、旧摘要、旧姓名、旧学号、旧日期等。
- **封面表格内容替换**：
  - Row 1（题目行）：替换为用户提供的论文题目
  - Row 3（课程名称）：替换为用户提供的课程名
  - Row 4（姓名+学号）：替换为用户提供的姓名和学号
  - Row 5（学院）：替换为用户提供的学院/院别
  - Row 6（时间）：自动填入当前系统日期
- **段落格式保留原则**：
  - 标题段落（引言、一级标题等）：保留原段落的 pPr，通常无首行缩进（firstLine=0）
  - 正文段落：保留原段落的 pPr，通常有首行缩进（firstLine=480）
  - **二级标题**：宋体小四加粗，有首行缩进（和正文一样）
  - 参考文献标题：使用标题段落模板（无首行缩进）
  - 参考文献条目：使用正文段落模板（有首行缩进）
- 如果模板首页有 Logo 图片段落，要保留或插回封面，不要在重写 XML 时丢失。
- 封面信息通过表格保留，日期自动填入表格最后一行。

如需自动生成 DOCX，可使用本技能的脚本。脚本会自动：
1. 识别模板中的 Logo、封面表格、标题段落、正文段落
2. 替换表格中的占位符内容
3. 保留原模板段落格式，不做强制覆盖
4. 自动填入当前日期

```bash
python /Users/gracegaoya/.claude/skills/buaa-final-assignment/scripts/build_from_template.py \
  --template “/Users/gracegaoya/.claude/skills/buaa-final-assignment/assets/templates/buaa-course-paper-template.docx” \
  --content paper.json \
  --output “课程名-学号-姓名-论文题目.docx”
```

`paper.json` 格式见脚本说明。若脚本不适合当前模板，仍按上述原则手工解析/替换 XML。

### 6. 文件命名

优先使用老师要求的命名格式。若没有明确要求，使用：

`课程名-学号-姓名-论文题目.docx`

文件名中去掉不适合文件系统的字符，如 `/ : * ? " < > |`，题目过长时保留核心短标题。

### 7. 验证清单

输出前检查：

- 文件能被解压并解析 `word/document.xml`。
- 文档仍包含图片 drawing 元素和 `word/media/` 文件，除非模板本来没有图片。
- 无模板提示文字、旧示例论文内容、旧姓名/学号/旧日期残留。
- **格式符合论文标准：标题黑体三号，摘要/关键词/正文宋体小四，1.5 倍行距，页边距常规。优先按模板内置设置执行。**
- **日期自动使用当前系统日期，格式为”YYYY年MM月DD日”，正确填入封面表格 Row 6。**
- **封面表格内容已替换**：题目、课程名、姓名、学号、学院等信息与用户提供的内容一致。
- **段落格式保留**：标题段落无首行缩进，正文段落有首行缩进，完全保留模板原 pPr。
- 中文字数达到要求。
- 包含摘要、关键词、引言、结论、参考文献。
- 正文引用编号与参考文献大体对应。
- 包含 AI 使用披露（如果课程要求披露或用户提到 AI）。
- 正文已通过中文学术改写流水线（11 步审计）：口语正式化、套话清理、客观转写、长句拆分、逻辑衔接、适度名词化、术语校准、冗余压缩、格式规范化、重点调整、语义核验。
- 封面个人信息已用用户提供内容填入表格；如果用户未提供，文档中会保留占位符，并在交付说明中列出需要用户后续补充的项目。
- 文件名符合老师要求或默认命名规则。

### 8. 给用户的交付说明

最终回复要简短说明：

- 输出文件路径。
- 已满足哪些要求。
- 是否仍有任何缺失个人信息。
- 提醒用户打开 Word 目视检查 Logo、封面和分页。

如果信息齐全，不要说“你还需要替换姓名学号”；只有在用户没提供时才提醒。

示例：

> 已生成：[文件名.docx](文件名.docx)。已使用内置北航结课作业模板，保留 Logo 和段落格式，中文约 xxxx 字，含参考文献和 AI 使用披露。封面信息已填入你提供的姓名、学号、学院和课程名；建议提交前打开 Word 目视检查分页。

## 常见错误

- 只“借用模板文件名”而没有保留模板样式。
- 重建 `document.xml` 时丢掉 Logo 或图片关系。
- 把模板提示文字或内置模板示例论文留在正文里。
- 非要等所有信息齐全才开始写，其实缺项可以先用占位符。
- 正文没有引用编号，只有文末参考文献。
- 文末没有 AI 使用披露。
- 文风过于模板化，堆“首先其次再次最后”。
- 用户已给姓名学号，却仍提醒用户自行替换。
- 提问方式太像填表格，应改成引导式、更自然的中文表述。
